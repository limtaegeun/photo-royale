- **브랜치**: `feat/p02-team-assignment` (전체 미커밋 — working tree 기준)

## 테스트 환경·사전 조건

- **계정**: 호스트 1 + 게스트 4 이상(**남2·여2 최소** — 혼성/이월 검증용. 홀수 검증 시 5명). 가입 시 성별 필수라 성별 미상은 스펙상 없음(방어 코드만 존재).
- **멀티 클라이언트**: 호스트(일반 창) + 게스트(시크릿/다른 브라우저 프로필). 실시간 판정은 항상 **상대 클라이언트 화면** 기준.
- **rules 검증**: Firestore 콘솔은 admin이라 rules가 적용되지 않음 — 거부 케이스는 **클라이언트 SDK 콘솔 직접 호출**(updateDoc)로 검증.
- **네트워크**: DevTools 오프라인 토글(확정 실패 재현), 스로틀.
- **기기**: 데스크톱 Chrome(기본) + 모바일 뷰포트. 실기기(iOS Safari/Android Chrome)는 waiting-room.md M군에서 커버된 기반 위에 배정 화면 스팟.
